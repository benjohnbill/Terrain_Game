# Baseline: stack-literacy starting point

Established from the handoff doc (disclosed prior knowledge, not yet
demonstrated): the learner can build features in vanilla JS inside this repo but
lacks a layered mental model of technology stacks, which is why stack-change
questions recur as anxiety rather than analysis.

Future sessions start from "can code, cannot yet classify or decide stack
changes." The first lesson is therefore the four-layer model, grounded in this
repo's real files. The bar for success is **re-derivation, not recall** — the
learner should be able to reconstruct ADR 0016's reasoning, not just repeat its
conclusion.
