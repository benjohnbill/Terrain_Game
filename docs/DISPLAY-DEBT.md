# Display-Debt Register

The single parking place for UI / display-design debts — reads, cards,
and surfaces the sealed mechanics call for but whose visual design is
deliberately deferred until B (the playable slice / terrain-cradle map)
gives them a real UI to live in. Symmetric with `SYNC-DEBT.md`
(doc-sync debts) but for *display* design, not documentation.

**Registration only.** Each row records WHAT read is owed and WHERE its
mechanic was sealed — the design (layout, interaction, copy) is B's UI
work, not authored here. Birthplace stays authoritative: the mechanic's
definition lives in its GLOSSARY/RULINGS/ADR home; this register points,
never redefines. Seeded 2026-07-06 (A-4 B6) from the match-arc riders
and the A-3 economy rider.
